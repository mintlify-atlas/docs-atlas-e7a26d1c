## Overview

Debentialc provides extensive PlaceholderAPI integration for displaying booster information, player stats, and server data in chat, scoreboards, holograms, and other plugins.

<Note>
  **Requirement**: PlaceholderAPI must be installed on your server. Download from [SpigotMC](https://www.spigotmc.org/resources/placeholderapi.6245/).
</Note>

## Installation

1. Install PlaceholderAPI on your server
2. Install Debentialc plugin
3. Restart server - expansions register automatically
4. Verify with `/papi list` - you should see `boosters` and `debentialc`

## Available Expansions

### Boosters Expansion (`%boosters_<placeholder>%`)

#### Global Booster Placeholders

<AccordionGroup>
  <Accordion title="Global Booster Status">
    | Placeholder | Description | Example Output |
    |-------------|-------------|----------------|
    | `%boosters_global_active%` | Global booster status | `§aActivo` or `§cInactivo` |
    | `%boosters_global_multiplier%` | Current multiplier | `§62.50x` |
    | `%boosters_global_percentage%` | Bonus percentage | `§a+150%` |
    | `%boosters_global_time%` | Time remaining | `§61h 30m 45s` |
    | `%boosters_global_activator%` | Who activated it | `§6AdminName` |
    
    **Source**: `BoosterPlaceholder.java:48-67`
    
    **Usage example**:
    ```yaml
    # In DeluxeMenus
    display_name: "Global Booster: %boosters_global_active%"
    lore:
      - "Multiplier: %boosters_global_multiplier%"
      - "Bonus: %boosters_global_percentage%"
      - "Time: %boosters_global_time%"
      - "By: %boosters_global_activator%"
    ```
  </Accordion>

  <Accordion title="Personal Booster Status">
    | Placeholder | Description | Example Output |
    |-------------|-------------|----------------|
    | `%boosters_personal_active%` | Personal booster status | `§aActivo` or `§cInactivo` |
    | `%boosters_personal_multiplier%` | Personal multiplier | `§61.50x` |
    | `%boosters_personal_percentage%` | Personal bonus | `§a+50%` |
    | `%boosters_personal_level%` | Booster tier name | `§6Legendary` |
    | `%boosters_personal_time%` | Time remaining | `§614m 32s` |
    | `%boosters_personal_count%` | Available boosters | `§63` |
    
    **Source**: `BoosterPlaceholder.java:69-108`
    
    **Usage example**:
    ```yaml
    # In FeatherBoard scoreboard
    - "Personal Booster"
    - "Status: %boosters_personal_active%"
    - "Level: %boosters_personal_level%"
    - "Bonus: %boosters_personal_percentage%"
    - "Time: %boosters_personal_time%"
    ```
  </Accordion>

  <Accordion title="Combined Booster Information">
    | Placeholder | Description | Example Output |
    |-------------|-------------|----------------|
    | `%boosters_combined_multiplier%` | Total multiplier (global × personal) | `§63.75x` |
    | `%boosters_combined_percentage%` | Total bonus percentage | `§a+275%` |
    | `%boosters_any_active%` | Any booster active | `§aActivo` |
    
    **Source**: `BoosterPlaceholder.java:109-128`
    
    **Formula**: Combined multiplier = Global × Personal
    
    **Example calculation**:
    - Global: 2.5x
    - Personal: 1.5x
    - Combined: 2.5 × 1.5 = 3.75x
    - Percentage: (3.75 - 1.0) × 100 = +275%
    
    **Usage example**:
    ```yaml
    # In chat format plugin
    format: "[%boosters_combined_multiplier%] {USERNAME}: {MESSAGE}"
    ```
  </Accordion>
</AccordionGroup>

## Placeholder Behavior

### Player-Specific Placeholders

<Warning>
  Some placeholders require a player context and return default values when used globally:
  
  - `personal_*` placeholders return `§cInactivo` or `§7Ninguno`
  - `combined_*` placeholders use only global multiplier
  - `level`, `tps`, `race` return `0` or empty string
</Warning>

**Source reference**: `BoosterPlaceholder.java:136-144`

### Time Formatting

Time placeholders automatically format based on duration:

```
Hours remaining:   "2h 30m 15s"
Minutes remaining: "45m 20s"
Seconds remaining: "30s"
Expired:           "§cExpirado"
```

**Source**: `BoosterPlaceholder.java:148-165`

### Color Codes

Placeholders include color codes:

- `§a` - Green (active, positive)
- `§c` - Red (inactive, negative)
- `§6` - Gold (values, numbers)
- `§7` - Gray (none, zero)
- `§b` - Aqua (personal)
- `§e` - Yellow (emphasis)

## Troubleshooting

<AccordionGroup>
  <Accordion title="Placeholders show as text">
    **Problem**: `%boosters_global_active%` displays literally
    
    **Solutions**:
    1. Verify PlaceholderAPI is installed: `/papi list`
    2. Check expansion is registered: `/papi info boosters`
    3. Restart server after installing Debentialc
    4. Use `/papi parse me <placeholder>` to test
    
    **Example**:
    ```
    /papi parse me %boosters_global_multiplier%
    ```
  </Accordion>

  <Accordion title="Placeholders return empty or default values">
    **Problem**: Placeholders return `0`, `§cInactivo`, or empty strings
    
    **Common causes**:
    1. No booster is currently active
    2. Using personal placeholder without player context
    3. CustomNPCs not installed (for DBC placeholders)
    4. Player not loaded in DBC system
    
    **Verification**:
    ```
    /booster info
    /papi parse <player> %boosters_personal_active%
    ```
  </Accordion>

  <Accordion title="Booster expansion not found">
    **Problem**: `/papi info boosters` shows "Expansion not found"
    
    **Solutions**:
    1. Verify Debentialc is loaded: `/plugins`
    2. Check console for registration errors
    3. Ensure PlaceholderAPI loads before Debentialc
    4. Try manual registration: `/papi register boosters`
    
    **Check logs for**:
    ```
    [Boosters] PlaceholderAPI expansion registered
    ```
  </Accordion>

  <Accordion title="Colors not displaying">
    **Problem**: Color codes show as `§a`, `§6`, etc.
    
    **Solution**: The receiving plugin must support color codes. Use:
    - `ChatColor.translateAlternateColorCodes()` in Java
    - Built-in translation in most modern plugins
    - Replace `§` with `&` if plugin expects alternate codes
  </Accordion>
</AccordionGroup>

## Testing Placeholders

### Command Line Testing

```bash
# Test any placeholder

# Test another player

/papi parse me %boosters_global_active%
/papi parse me %boosters_personal_active%

```

## API for Developers

Integrate placeholders in your own plugins:

```java
import me.clip.placeholderapi.PlaceholderAPI;

String formatted = PlaceholderAPI.setPlaceholders(player, 
    "Your total multiplier: %boosters_combined_multiplier%"
);

player.sendMessage(formatted);
```
