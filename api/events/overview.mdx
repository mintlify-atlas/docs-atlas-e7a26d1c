The Debentialc plugin uses Bukkit's event system to respond to player actions, game events, and custom triggers. The plugin is organized into several modules, each with its own set of event listeners.

## Architecture

All event listeners in the plugin implement Bukkit's `Listener` interface and use the `@EventHandler` annotation to register event handling methods. Events are automatically registered when the plugin loads.

### Event Priority

Listeners use different event priorities to control execution order:

- `EventPriority.NORMAL` - Standard priority for most listeners
- `EventPriority.HIGH` - For listeners that need to run after normal handlers
- `EventPriority.MONITOR` - For read-only monitoring (not used for canceling)

@EventHandler(priority = EventPriority.NORMAL)
public void onPlayerJoin(PlayerJoinEvent event) {

    }
}

#### ConsumableItemListener

Handles right-click interactions with custom items that have consumable properties, commands, or TP values.

**Key Events:**
- `PlayerInteractEvent` - Processes item usage

**Features:**
- Executes console commands with placeholder support (`@dp` for user, `@p` for target)
- Grants Training Points (TP) to players
- Handles stack consumption
- Supports target detection (looks at player)

**Command Placeholders:**
- `@dp` - The player using the item
- `@p` - The player being looked at (or self if none)

**Example:**
```java
@EventHandler
public void onItemUse(PlayerInteractEvent event) {
    ItemStack item = event.getPlayer().getItemInHand();
    CustomItem customItem = itemCmd.toItemCustom(item);
    
    if (customItem != null && customItem.isConsumable()) {
        // Execute commands
        if (customItem.getCommands() != null) {
            executeCommands(player, customItem.getCommands());
        }
        
        // Grant TP
        if (customItem.getTpValue() > 0) {
            giveTP(player, customItem.getTpValue(), amount);
        }
        
        // Consume item
        item.setAmount(item.getAmount() - 1);
    }
}
```

#### ScriptExecutionListener

Executes JavaScript scripts associated with custom items when they are used.

**Key Events:**
- `PlayerInteractEvent` (HIGH priority) - Detects item usage and executes scripts

**Features:**
- Creates script execution context with player, item, location, and world data
- Integrates with ScriptManager for script execution
- Supports event type identification

**Script Context:**
```java
ScriptContext context = new ScriptContext(player)
    .setItem(item)
    .setLocation(player.getLocation())
    .setWorld(player.getWorld())
    .setEventType("use");

ScriptManager.getInstance().executeScript(itemId, context);
```

#### ArmorEquipEvent

Custom event that fires when a player equips or unequips armor pieces.

**Event Properties:**
- `player` (Player) - The player equipping armor
- `newItem` (ItemStack) - The armor piece being equipped
- `cancelled` (boolean) - Whether the event is cancelled

**Usage:**
```java
@EventHandler
public void onArmorEquip(ArmorEquipEvent event) {
    Player player = event.getPlayer();
    ItemStack armor = event.getNewItem();
    
    // Check if it's a custom armor piece
    if (isCustomArmor(armor)) {
        // Apply stat bonuses
        applyArmorStats(player, armor);
    }
}
```

#### NPCDeathListener

The primary event handler for raid progression. Tracks NPC deaths, manages wave completion, and handles rewards.

**Key Events:**
- `INpcEvent.DiedEvent` (CustomNPCs API) - Fired when an NPC dies

**Features:**
- Tracks killed NPCs per wave
- Updates wave progress in real-time
- Manages wave countdown timers
- Handles wave completion and rewards
- Manages raid completion and victory
- Provides player feedback (messages, sounds, effects)

## Creating Custom Event Listeners

To create your own event listener that integrates with the Debentialc plugin:

### Step 1: Implement Listener Interface

```java
import org.bukkit.event.Listener;
import org.bukkit.event.EventHandler;
import org.bukkit.event.player.PlayerInteractEvent;

public class MyCustomListener implements Listener {
    
    @EventHandler
    public void onPlayerInteract(PlayerInteractEvent event) {
        // Your event handling logic
    }
}
```

### Step 2: Register the Listener

In your main plugin class or module:

```java
import org.bukkit.Bukkit;
import org.bukkit.plugin.java.JavaPlugin;

public class MyPlugin extends JavaPlugin {
    
    @Override
    public void onEnable() {
        // Register listeners
        Bukkit.getPluginManager().registerEvents(new MyCustomListener(), this);
    }
}
```

### Step 3: Use Event Priority

```java
@EventHandler(priority = EventPriority.HIGH)
public void onHighPriorityEvent(PlayerInteractEvent event) {
    // Runs after NORMAL priority handlers
}

@EventHandler(priority = EventPriority.MONITOR, ignoreCancelled = true)
public void onMonitorEvent(PlayerInteractEvent event) {
    // For read-only monitoring, doesn't modify event
}
```

## Best Practices

<Tip>
**Performance:** Avoid heavy operations in event handlers. Use async tasks or schedulers for expensive operations.
</Tip>

<Tip>
**Error Handling:** Always use try-catch blocks to prevent one event failure from breaking others.
</Tip>

<Warning>
**Event Cancellation:** Be careful when cancelling events, as it may affect other plugins or listeners.
</Warning>

### Example: Async Processing

```java
@EventHandler
public void onExpensiveEvent(PlayerInteractEvent event) {
    Player player = event.getPlayer();
    
    // Run expensive operation async
    Bukkit.getScheduler().runTaskAsynchronously(plugin, () -> {
        // Heavy database query or calculation
        String result = performExpensiveOperation();
        
        // Return to main thread for game interaction
        Bukkit.getScheduler().runTask(plugin, () -> {
            player.sendMessage(result);
        });
    });
}
```

### Example: Safe Event Handling

```java
@EventHandler
public void onPlayerEvent(PlayerInteractEvent event) {
    try {
        Player player = event.getPlayer();
        ItemStack item = player.getItemInHand();
        
        if (item == null || item.getTypeId() == 0) {
            return;
        }
        
        // Your logic here
        
    } catch (Exception e) {
        plugin.getLogger().severe("Error handling player event: " + e.getMessage());
        e.printStackTrace();
    }
}
```
