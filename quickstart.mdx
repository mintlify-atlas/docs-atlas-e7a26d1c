# Quick Start Guide

This guide will get you up and running with Debentialc's four major systems: Boosters, Custom Items, Raids, and Territory Claims.

<Note>
  Ensure you've completed [Installation](/installation) before proceeding.
</Note>

## Basic Workflow

<Steps>
  <Step title="Verify Installation">
    Check that Debentialc is loaded:
    ```bash
    /plugins
    ```
    Look for **Debentialc** in green.
  </Step>
  
  <Step title="Check Permissions">
    Ensure you have admin permissions:
    ```bash
    /lp user <your_name> permission set debentialc.* true
    ```
    Or use your permissions plugin's equivalent.
  </Step>
  
  <Step title="Explore Help Commands">
    ```bash
    /booster help
    /ci help
    /terrain help
    /raid help
    ```
  </Step>
</Steps>

## Booster System Quick Start

The booster system provides global and personal multipliers for gameplay bonuses.

### Activating a Global Booster

<CodeGroup>
```bash Admin Commands
# Activate 50% bonus for 1 hour
/booster global activate 50% 1h

# Activate 150% bonus for 2.5 hours
/booster global activate 150% 2h30m

# Check active booster
/booster global info

# Deactivate early
/booster global deactivate
```

```java Programmatic Activation
import org.debentialc.boosters.managers.GlobalBoosterManager;
import org.bukkit.Bukkit;

// Activate 2.5x global booster for 1 hour
GlobalBoosterManager.activateBooster(2.5, "AdminName", 3600);

// Broadcast to all players
Bukkit.broadcastMessage("§6§lBOOSTER ACTIVATED: 2.5x for 1 hour!");
```
</CodeGroup>

See BoosterCommand.java:77-125 for implementation details.

### Giving Personal Boosters

<Steps>
  <Step title="Give Booster to Player">
    ```bash
    /booster personal add <player> <level>
    ```
    Example:
    ```bash
    /booster personal add Steve 3
    ```
    
    Levels 1-5 (configured in BoosterSettings):
    - Level 1: +15% bonus
    - Level 2: +30% bonus
    - Level 3: +60% bonus
    - Level 4: +120% bonus
    - Level 5: +250% bonus
  </Step>
  
  <Step title="Player Activates Booster">
    Player runs:
    ```bash
    /booster personal list
    /booster personal activate 0
    ```
    
    The index (0, 1, 2...) corresponds to the booster in their inventory.
  </Step>
  
  <Step title="Check Combined Multiplier">
    ```bash
    /booster info
    ```
    
    Shows both global and personal boosters active.
  </Step>
</Steps>

See BoosterCommand.java:159-282 for personal booster logic.

### Using Boosters in Your Code

```java
import org.debentialc.boosters.core.BoosterUtils;
import org.bukkit.entity.Player;

public void applyBoosterToReward(Player player, double baseReward) {
    // Get combined multiplier (global + personal + rank)
    double multiplier = BoosterUtils.calculateCombinedMultiplier(player.getUniqueId());
    
    // Apply to calculation
    double finalReward = baseReward * multiplier;
    
    player.sendMessage("§aBase reward: " + baseReward);
    player.sendMessage("§eMultiplier: " + BoosterUtils.formatMultiplier(multiplier));
    player.sendMessage("§6Final reward: " + finalReward);
}
```

See Main.java:224-231 for example usage.

## Custom Items Quick Start

Create powerful custom items with stats, effects, and bonuses.

### Creating Your First Custom Item

<Steps>
  <Step title="Hold Base Item">
    Hold a diamond sword in your hand (or any item).
  </Step>
  
  <Step title="Rename and Add Lore">
    Use an anvil or commands:
    ```bash
    /ci create my_sword
    /ci rename §6§lLegendary Blade
    /ci addline §7A weapon of immense power
    /ci addline §a+50 Strength | +25% Dexterity
    ```
  </Step>
  
  <Step title="Add Stat Bonuses">
    ```bash
    # Add +50 flat strength
    /ci plus my_sword 50 str
    
    # Add +25% dexterity multiplier
    /ci percentage my_sword 0.25 dex
    
    # Add -10 willpower penalty
    /ci less my_sword 10 will

  </Step>
  
  <Step title="Add Special Effects">

    # 5% health regeneration per tick
    /ci effect HEALTHREGEN 0.05
    
    # 3% ki regeneration
    /ci effect KIREGEN 0.03
    ```
  </Step>
  
  <Step title="Give to Players">
    ```bash
    /ci give my_sword PlayerName 1
    ```
  </Step>
</Steps>

## Territory Claims Quick Start

Create purchasable territories with protection and member management.

### Admin: Creating a Territory

<Steps>
  <Step title="Define Territory">
    ```bash
    /terrain create <id> <chunks>
    ```
    Example:
    ```bash
    /terrain create plaza 3
    ```
    
    This creates a 3x3 chunk area (48x48 blocks).
  </Step>
  
  <Step title="Set Purchase Price">
    ```bash
    /terrain price plaza 25000
    ```
    
    Price is in your server's economy currency (requires Vault).
  </Step>
  
  <Step title="Generate in World">
    Stand at the desired location:
    ```bash
    /terrain commit plaza
    ```
    
    This creates visual borders and protection at your current position.
  </Step>
  
  <Step title="Verify Creation">
    ```bash
    /terrain info plaza
    /terrain list
    ```
  </Step>
</Steps>

See TerrainCommand.java:71-163 for creation logic.

### Player: Purchasing a Territory

<Steps>
  <Step title="Find Available Territory">
    Walk to a territory with a sign marker, or:
    ```bash
    /terrain list
    ```
  </Step>
  
  <Step title="Check Territory Info">
    ```bash
    /terrain info
    ```
    
    Shows price, size, and availability.
  </Step>
  
  <Step title="Purchase Territory">
    Right-click the territory sign or use the purchase command (implementation-specific).
    
    Your account is charged automatically via Vault.
  </Step>
  
  <Step title="Manage Your Territory">
    ```bash
    # Add a trusted member
    /terrain member plaza FriendName all
    
    # Add a builder
    /terrain member plaza BuilderName build
    
    # Remove a member
    /terrain kick plaza OldFriend
    
    # Transfer ownership
    /terrain transfer plaza NewOwner
    
    # Sell back (50% refund)
    /terrain sell plaza
    ```
  </Step>
</Steps>

See TerrainCommand.java:192-342 for member management.

### Reward System Integration

```java
import org.debentialc.boosters.core.BoosterUtils;
import org.bukkit.entity.Player;

public void giveQuestReward(Player player, double baseXP) {
    // Apply booster multiplier
    double multiplier = BoosterUtils.calculateCombinedMultiplier(player.getUniqueId());
    double finalXP = baseXP * multiplier;
    
    // Give reward
    player.giveExp((int) finalXP);
    
    // Inform player
    if (multiplier > 1.0) {
        player.sendMessage("§aReward: " + finalXP + " XP (" 
            + BoosterUtils.formatPercentage(multiplier) + " boost!)");
    }
}
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Booster not applying to calculations">
    Ensure you're using the API correctly:
    
    ```java
    // CORRECT
    double mult = BoosterUtils.calculateCombinedMultiplier(player.getUniqueId());
    double result = baseValue * mult;
    
    // INCORRECT - only gets global
    double globalOnly = GlobalBoosterManager.getCurrentMultiplier();
    ```
    
    See Main.java:224-231 for proper usage.
  </Accordion>
  
  <Accordion title="Custom items losing stats">
    Stats are tied to the item's display name and lore. If players rename items via anvil, stats may not apply.
    
    **Solution**: Use the `CustomItemCommand.isCustom()` method to validate (CustomItemCommand.java:398-411):
    
    ```java
    public boolean isCustom(ItemStack item) {
        if (item == null || !item.hasItemMeta()) return false;
        ItemMeta meta = item.getItemMeta();
        return items.containsValue(
            new CustomItem()
                .setMaterial(item.getTypeId())
                .setLore(meta.getLore())
                .setDisplayName(meta.getDisplayName())
        );
    }
    ```
  </Accordion>
  
  <Accordion title="Territory borders not appearing">
    Ensure the terrain was committed:
    
    ```bash
    /terrain list
    # Look for territories marked "(sin generar)"
    
    /terrain commit <id>
    ```
    
    Stand at the desired location before committing.
  </Accordion>
  
  <Accordion title="Raid NPCs not triggering events">
    Verify CustomNPCs API is available:
    
    1. Check console for:
       ```
       [Debentialc] [Raids] NPCDeathListener registered correctly
       ```
    
    2. If not registered:
       - Ensure CustomNPCs loads before Debentialc
       - Check CustomNPCs version compatibility
       - Verify API is exposed (Main.java:107-110)
  </Accordion>
</AccordionGroup>
